📦 O que é um Token?

Um token é a unidade mínima de texto que o modelo processa. Não é necessariamente uma palavra inteira — pode ser um pedaço de palavra, um sinal de pontuação, um espaço, ou até um único caractere.

O modelo GPT-4 usa um vocabulário de ~100.000 tokens (cl100k_base). Cada token tem um ID numérico único — é exatamente esse número que entra na rede neural, não o texto em si.

Exemplo: a palavra incrível pode virar 2 tokens: incr + ível. Já cat em inglês é 1 token só. Isso explica por que textos em português são mais "caros" (mais tokens) que em inglês.

Regra prática: 1 token ≈ 4 caracteres em inglês, ≈ 3 caracteres em português. 100 tokens ≈ 75 palavras em inglês.

⚙️ Como funciona o BPE (Byte-Pair Encoding)?

O BPE é o algoritmo usado para criar o vocabulário de tokens. Funciona assim:

1. Começa com caracteres individuais — cada letra/símbolo é um token separado.

2. Encontra os pares mais frequentes — se "a" + "b" aparecem juntos o tempo todo, junta em "ab".

3. Repete até o vocabulário ter o tamanho desejado (~100k iterações para o GPT-4).

Resultado: palavras comuns ficam inteiras (1 token), palavras raras são quebradas em partes reconhecíveis. O modelo nunca encontra uma palavra "desconhecida" — ele sempre consegue representá-la em partes.

Observe no painel ao lado: palavras como transformação são divididas em transfor + mação. Cada parte recebe um ID diferente. O modelo aprende o significado de cada pedaço e como combiná-los.

👁️ Self-Attention — Como o modelo entende contexto

O mecanismo de Self-Attention é o coração do Transformer. Ele permite que cada token "olhe" para todos os outros tokens da frase e decida o quanto de atenção dar a cada um.

A fórmula é: Attention(Q, K, V) = softmax( Q · Kᵀ / √d ) · V

• Q (Query): "O que eu estou procurando?"
• K (Key): "O que eu tenho a oferecer?"
• V (Value): "O que eu passo adiante se for relevante?"

No heatmap, cada linha é um token fazendo a pergunta; cada coluna é um token sendo consultado. Uma célula mais brilhante significa que o token-linha presta mais atenção ao token-coluna.

Isso é o que permite o modelo entender que em "O banco estava cheio", a palavra "banco" deve se relacionar com "pagamento" (banco financeiro) — o contexto muda o significado!

🌐 PT vs EN — Por que o português usa mais tokens?

O GPT-4 foi treinado com muito mais texto em inglês do que em português. Por isso o vocabulário BPE foi otimizado para o inglês — palavras comuns em inglês cabem em 1 token, enquanto equivalentes em português são fragmentadas.

Exemplo comparativo:

• EN: transformation → 1 token  |  PT: transformação → 3 tokens

• EN: beautiful → 2 tokens  |  PT: linda → 2 tokens (empate!)

• EN: the cat sat → 3 tokens  |  PT: o gato sentou → 4–5 tokens

Consequência prática: enviar o mesmo conteúdo em português custa ~30-50% mais tokens do que em inglês — e os modelos cobram por token. Além disso, o modelo "vê" o texto português como fatias menores, o que pode afetar a qualidade da compreensão.

🔢 Como números e código são tokenizados?

Números são tokenizados de forma imprevisível — esse é um dos motivos pelos quais LLMs têm dificuldade com aritmética!

• 1234 pode ser 1 token, mas 12345 pode ser 2 ou 3 tokens diferentes.

• O modelo não "vê" dígitos individuais — ele vê pedaços de número, sem noção de posição decimal ou valor real.

Para código, o tokenizador foi ajustado para reconhecer padrões comuns de programação. Palavras-chave como def, for, import são 1 token. Nomes de variáveis longos são quebrados. Espaços de indentação também são tokens!

Experimente digitar um número grande como 3141592653589793 ou um trecho de código Python no campo "Texto Livre" e observe como o tokenizador lida com isso.
